# ADR 0002: Syncing app codes

## Status

Accepted

## Context

Explain the problem, constraints, and forces that required this decision.  
What triggered it? What options were considered?

The project has App codes which need a human readabme message in the response.
it would be tedious and unmaintable to add the code and its corresponding message all over the API endpoint.

## Decision

State the decision that was made.  
Be clear and direct.

every API endpoint will always have a code field in the response. if it's missing, the API will send an error in the console and not send the message.

The middleware will read the response and the corresponding message field automatically.

## Consequences

Positive consequences: message fields wont need to be rewritten everywhere.
Negative consequences: Every response has to do O(n) comparisons when searching for a response. This is not too bad because the App codes can be seperated through their type (e.g. AUTH, VAL), reducing the search time drastically.
