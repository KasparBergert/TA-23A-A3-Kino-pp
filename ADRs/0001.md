# ADR 0001: Use third-party API for films

## Status

Rejected

## Context

Maintaining up-to-date film data in-house is resource-intensive. A third-party API could provide latest films, release dates, popularity scores, and metadata useful for ticket sales.

## Decision

We will not integrate a third-party film API at this stage. The integration and maintenance effort does not justify the value gained.

## Consequences

Positive:

- No dependency on an external service.
- Database integrity preserved; only internal data flows.

Negative:

- Film data must be updated manually.
- Lack of automatic freshness for film metadata.

## Alternatives Considered

- **Frontend-only API integration** — rejected because seat, showroom, and schedule data must be tied directly to films in backend business logic.
- **Automated scripts for data ingestion** — rejected due to integration overhead versus current value. Can be reconsidered later if manual updates become unmanageable.
