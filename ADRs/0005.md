# ADR 0005: Remove `users_roles` Table

## Status  
Accepted

## Context  
The `users_roles` join table allowed users to have multiple roles, which contradicts the rule that each user can have only one role.

## Decision  
Deleted `users_roles` and added a `role_id` foreign key in `users` referencing `roles.id`, enforcing a 1:N relationship (one role → many users).

## Consequences  
- **Positive:** Simpler schema and queries, clearer role constraint.  
- **Negative:** Removes ability to assign multiple roles per user (not required).  

## Alternatives  
None — the previous design was incorrect at the database level.
