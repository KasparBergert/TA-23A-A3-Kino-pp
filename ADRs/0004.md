# ADR 0004: give seats table an is_available field

## Status

Accepted

## Context

with the current way the table is represented, no grid can be made out of it because the leftmost top and the rightmost bottom seats cannot be selected, thus making a grid
impossible to make.
secondly, currently 2 fetches have to occur to get the is_available seats with the prisma ORM, that is not ideal.

## Decision

add field is_available to the seat table.
add all seats as rows in the hall.


## Consequences

Positive consequences: A grid can now be made on the frontpage of the available seats. and it's easier to fetch the available seats by just running count().
Negative consequences: if there are thousands of seats, then the server will slow down because of how many times it has to count the seats.

## Alternatives Considered

- Create triggers for the seat availability: simply put a counter for the halls for how many seats are available. But that is technically not the phisical property of a hall.
